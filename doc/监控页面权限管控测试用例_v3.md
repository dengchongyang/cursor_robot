# 监控页面权限管控测试用例 (v3-严格缩进版)

## 用例

### 准入控制-未登录访问-验证整页底部不可见及弹窗触发
- 1.准备一个有效的监控页面链接
  2.确保当前浏览器处于未登录状态
        - 1.在浏览器地址栏输入监控链接并回车
          2.观察页面底部区域（在线信息栏、菜单列表）
          3.观察页面中心区域
                - 1.页面底部区域完全隐藏，不可见
                  2.页面自动弹出“账号登录”弹窗

### 准入控制-弹窗关闭-验证底部区域持续隐藏
- 1.处于未登录状态并已进入监控页面
  2.登录弹窗已显示
        - 1.点击登录弹窗右上角的关闭按钮“X”
          2.再次观察页面底部区域
                - 1.登录弹窗消失
                  2.页面底部区域依然保持隐藏状态，无法查看任何参会数据

### 登录交互-正确登录-验证底部区域显示及数据加载
- 1.处于未登录状态，登录弹窗已弹出
  2.准备一组正确的账号密码
        - 1.在弹窗中输入正确的账号和密码
          2.点击“登录”按钮
                - 1.登录成功，弹窗自动消失
                  2.页面底部区域立即显示
                  3.底部各菜单列表正常加载并展示参会信息

### 登录交互-错误登录-验证底部区域持续隐藏
- 1.处于未登录状态，登录弹窗已弹出
        - 1.在弹窗中输入错误的账号或密码
          2.点击“登录”按钮
                - 1.页面提示“账号或密码错误”
                  2.登录弹窗不消失
                  3.页面底部区域持续保持隐藏状态

### 状态持久化-页面刷新-验证已登录状态免登录
- 1.已成功登录监控页面
  2.底部区域当前可见
        - 1.点击浏览器刷新按钮
                - 1.页面重新加载后，底部区域直接显示
                  2.不弹出登录弹窗

### 安全性-登录态失效-验证底部区域自动隐藏
- 1.已成功登录监控页面，底部区域可见
        - 1.在后端手动清除该 Session 或等待 Token 过期
          2.在监控页面执行任意操作或刷新页面
                - 1.页面底部区域立即变为隐藏状态
                  2.页面自动弹出登录弹窗

### 安全性-前端篡改-验证接口权限校验
- 1.处于未登录状态，底部区域已通过 CSS 隐藏
        - 1.通过开发者工具手动修改 CSS，将隐藏区域设为可见 (`display: block`)
          2.查看列表内的数据内容
                - 1.即使区域显示，列表内应无实际数据（接口返回权限错误）
                  2.确保敏感会议数据不外泄
