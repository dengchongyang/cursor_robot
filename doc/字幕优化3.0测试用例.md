# 字幕优化 3.0 详细测试用例

## 用例

### 运营后台-新建/编辑路演-验证日语实时字幕开启逻辑
- 1.进入路演新建或编辑页面
  2.找到会议原语言设置项
- 1.将会议原语言选择为“日语”
  2.尝试开启“实时字幕”开关
- 1.实时字幕开关可正常开启
  2.保存后，该路演具备日语实时字幕能力

### 运营后台-新建/编辑路演-验证自动识别提示
- 1.进入路演新建或编辑页面
- 1.将会议原语言选择为“自动识别”
- 1.页面红字提示：“自动识别仅支持中文和英文识别，若为日语会议，请选择【日语】选项”

### 运营后台-字幕翻译设置-验证三语互译选项显示
- 1.开启实时字幕开关
- 1.查看字幕翻译开关下的选项列表
- 1.固定显示“中文”、“英文”、“日语”三个选项
  2.“中文”和“英文”默认处于勾选状态

### 运营后台-字幕翻译设置-验证不同原语言下的翻译可选性
- 1.分别设置会议原语言为“中文”、“英文”、“日语”、“自动识别”
- 1.在每种原语言下，查看字幕翻译的可勾选列表
- 1.原语言为“自动识别”：支持勾选“中文”、“英文”、“日语”
  2.原语言为“中文”：支持勾选“英文”、“日语”
  3.原语言为“英文”：支持勾选“中文”、“日语”
  4.原语言为“日语”：支持勾选“中文”、“英文”

### Web端-字幕交互-验证开关与设置拆分
- 1.路演已开启字幕翻译功能
  2.用户进入路演厅
- 1.点击底部字幕 icon
  2.鼠标悬浮在字幕显示区域
  3.点击区域内的【设置】按钮
- 1.点击 icon 正常开启/关闭字幕（默认显示原文）
  2.悬浮时显示【设置】和【关闭】按钮
  3.点击【设置】打开字幕设置弹窗，支持独立勾选原文及译文语言

### Web端-字幕显示-验证双语显示逻辑
- 1.打开字幕设置弹窗
- 1.同时勾选“原文”和其中一种“译文”
- 1.字幕区域同时显示两行字幕（原文在前，译文在后）

### 移动端-字幕交互-验证翻译开启时的弹窗逻辑
- 1.后台已勾选字幕翻译选项
  2.用户使用移动端进入路演厅
- 1.点击底部字幕开关 icon
- 1.直接打开字幕设置弹窗（而非直接开关）
  2.弹窗内包含原文开关、译文单选列表及“开启字幕/保存设置”按钮

### 移动端-字幕交互-验证已开启状态下的重复点击
- 1.字幕已处于开启状态（显示双语）
- 1.再次点击底部字幕开关 icon
- 1.依然打开字幕设置弹窗（重阳确认：移动端点击开关始终打开设置弹窗）

### 移动端-字幕交互-验证至少显示一种语言
- 1.在设置弹窗中取消勾选所有选项
  2.点击【开启字幕】或【保存设置】
- 1.页面弹出 Toast 提示：“至少显示一种语言的字幕”
  2.操作不生效，弹窗不关闭

### 全端-入会默认显示-验证语言匹配逻辑
- 1.后台勾选了【用户入会默认显示】
  2.全局语言设置为日语
  3.后台仅勾选了中英翻译（未勾选日语翻译）
- 1.用户进入路演厅
- 1.默认显示“原文（日语）+ 默认勾选的中文”（重阳确认：若无对应语言则取原文+默认中文）

### 后端-翻译任务-验证中途开启翻译逻辑
- 1.会议正在进行中，原本未勾选“日语”翻译
- 1.在运营后台中途勾选“日语”翻译并保存
- 1.后端立即启动翻译任务
  2.前端后续产生的消息开始显示日语译文
  3.历史已产生的消息不会补发日语翻译（重阳确认：默认这次就不会再有了）

### 回放字幕-功能一致性-验证日语及互译支持
- 1.路演已结束并生成回放
  2.路演配置了日语字幕及翻译
- 1.进入回放页面
  2.尝试切换字幕语言
- 1.回放字幕选项与实时字幕保持一致
  2.支持在播放过程中动态切换“原文/译文”（重阳确认：支持动态切换）
